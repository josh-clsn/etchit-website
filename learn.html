How to use etchit

Read this once. Try the walkthrough. There are four things the app does — etch, fetch, backup, sync — and a couple of small costs to understand. That's it.

Beta · as-is etchit is beta software provided as-is. The Autonomi network and Arbitrum RPC are run by third parties — we don't guarantee uptime, data availability, or recoverability. Failed or partial transactions can cause loss of funds (gas or tokens spent without delivering an etch), and losing your wallet seed or backup passphrase means losing access to anything they protect. § 01 // The mental model

What etchit is.

etchit saves text and text-based files (notes, code, configs, docs) to a peer-to-peer network so they live forever, without an account, server, or middleman. Every save returns a 64-character address. The address is also the read-key — whoever has it can fetch what you saved.

Three things to keep in your head:

  • The network is public. What you etch publicly is visible to anyone with the address. There is no concept of "delete later".
  • Nobody can take it down. Not us, not a court, not a hosting provider — there is no host. The data is replicated across a peer-to-peer network and stays reachable as long as the network runs. Take this seriously: only etch content you have the right to publish. Once it's on the network, you can't pull it back.
  • You hold the keys. Your wallet is the only thing tying anything to "you". Lose the wallet and there is no recovery email or password reset — that's the trade-off for not having an account.

Reading is free. Writing costs a tiny one-time payment from your wallet, paid in ANT (the network's storage token). chain/it sync is separately paid in ETH on Arbitrum (a small gas fee). More on both below.

§ 02 // The four actions

Everything etchit does, in four cards.

If you understand these four flows, you understand the whole app. Each one works the same on Android and on the desktop client.

A // Public etch
Etch once.

Type something. Tap Etch to network. Sign once in your wallet. When the network finishes confirming, you get a 64-hex address.

The address is how you fetch this etch back from any device. It's also a public read-key — treat it like a secret if what you wrote is sensitive.

Cost: tiny one-time ANT chunk fee from your wallet. The first prompt asks you to approve a session budget (default 20 ANT) so subsequent etches don't ask again until it's spent.
B // Fetch
Fetch forever.

Paste a 64-hex address. Tap Fetch. Get the content back.

No wallet, no signing, no payment. As long as you have the address, fetching is open and free.

Cost: free. Reads don't pay.
C // Private + Backup
Encrypt, then backup.

A private etch encrypts on your device first, then stores only the encrypted blob on the network. By default the decryption key — the data map — lives only on this device's encrypted storage. Lose or wipe the device and the data is unrecoverable.

Back up to fix that. Tap "Back up to network", let the app generate a 6-word passphrase, write it down, sign one more etch. The whole private set is re-encrypted with that passphrase and uploaded. Now the passphrase is your recovery key — it lives wherever you put it (paper, password manager, head). On any device: backup address + passphrase = restored private etches.

Step-by-step
  1. Connect your wallet. Any Arbitrum One wallet — MetaMask Mobile pairs over QR.
  2. Tick "Private etch" in the etch panel, then type or paste your content and add a title.
  3. Tap Etch and approve the chunk-payment in your wallet. Encryption happens on the device before any byte leaves it.
  4. That's the private etch. The encrypted blob is on the network; the data-map (decryption key) is on this device only. Retrieving it later prompts your fingerprint / device PIN.
  5. Open Settings → Back up private etches. Tap "Generate strong passphrase" — 6 BIP-39 words. Write them down; there's no recovery if you lose them.
  6. Tap "Etch to network" and approve. Your whole private set re-encrypts with that passphrase and uploads as a single etch. Copy the backup address.
  7. Add the backup to chain/it (optional but recommended). Long-press the backup entry in Etch history → "Add to chainmarks". Now the backup address rides along with your wallet — no copy-paste between devices, no lost stickies. The passphrase you wrote down is still the only thing that decrypts it.
  8. To restore on a new device or after a wipe: install etchit → connect the same wallet → Sync chainmarks → tap the backup entry → enter the passphrase. Your private etches reappear. (No chain/it? Settings → Restore private etches → paste backup address + passphrase manually.)
Cost: same one-time ANT fee as a public etch — the backup is just one more etch, the bytes are encrypted.
D // chain/it sync
One list, all your devices.

chain/it stores your most important etch addresses on Arbitrum so you can reach them from any device — just connect the wallet you used to pay for the etches, tap Sync chainmarks, and your list reappears. No address book to email yourself, no notes-app entries to lose.

Each entry is encrypted with a key derived from a signature from that same wallet, then written as a tiny self-transaction. The chain sees encrypted bytes and your wallet address — nothing else.

No app installed? The web viewer at etchit.io/chainmarks reads the same data straight from a browser — connect the same wallet, view the same list.

Step-by-step
  1. Connect the wallet you used to pay for the etches you want indexed.
  2. Open chain/it → Sync chainmarks. Sign one derive message — that's your encryption key, derived from your wallet, never typed.
  3. Existing entries appear. First-time use: the list is empty. Otherwise your synced list reappears.
  4. Add an etch: long-press it in Etch history → "Add to chainmarks", or use + Add by address. Approve the tiny self-tx in your wallet.
  5. Switch device. Install etchit on desktop (or open etchit.io/chainmarks in a browser) → connect the same wallet → Sync. Same list.
  6. Optional — back up the chainmark key (Settings → Chainmark key → Show). Survives a full wipe and lets you read your chain/it given the key plus the wallet address, without the original wallet.
Cost: small Arbitrum gas fee per change (no ANT, this isn't on the Autonomi network). Your wallet shows the exact estimate before you sign.
§ 03 // Try it yourself

Walkthrough — try it yourself.

The fastest way to internalise the four actions is to do them once. You'll need a phone, a desktop or laptop, and a wallet (MetaMask Mobile is the easiest). Each step waits on a network or wallet confirmation, so pace yourself — durations vary with peer health and gas conditions.

  1. Install etchit on Android. Grab the latest APK from github.com/etchit-io/etchit/releases. (Desktop client is at github.com/etchit-io/etchit-desktop/releases.)
  2. Connect your wallet. Open Settings → Connect Wallet. Pair via QR with MetaMask Mobile. When the wallet prompts, approve Arbitrum One as the network.
  3. Etch your first thing. Tap Etch. Type hello from my phone. Tap Etch to network. Sign in your wallet. When the address appears, copy the 64 hex characters.
  4. Fetch from another device. Open the desktop client (or another phone). Tap Fetch, paste the address, tap go. Your text appears. No wallet needed for fetching.
  5. Turn on chain/it. Back on the phone, Settings → chain/it → Set up chain/it. Sign once. (This signature derives an encryption key — it's a sign-message, not a send-transaction.)
  6. Sync from desktop. Connect the same wallet on the desktop, expand chain/it, tap Sync chainmarks. The same list appears.
  7. Make a private etch + back it up. On the etch panel, toggle private first, then type something secret and tap Etch. Once it's stored, tap Back up to network, let the app generate a 6-word passphrase, write it down, tap encrypt + etch.
  8. Restore on a different device. On the desktop (or a fresh install), Settings → Restore from backup, paste the backup address, type the 6-word passphrase into the boxes, tap Restore. Your private etches return.
§ 04 // The bill

What things cost.

Two kinds of payment, two different reasons. Storage on the Autonomi network is paid in ANT. chain/it sync is paid in ETH on Arbitrum (gas). Reading anything is free.

Action Pays in Goes to Roughly
Public etch ANT network nodes that store your chunks one-time chunk fee
Private etch ANT same as public — bytes are encrypted client-side same as public
Backup to network ANT network nodes — backup is just another etch same as public
Fetch (anything) free — nothing
chain/it sync ETH Arbitrum block producers (gas) tiny per-tx fee

Your wallet always shows the exact estimate before you sign. If a chain/it sync looks more expensive than you expected, feel free to wait and try again later — the entry stays in your queue locally.

§ 05 // Things break sometimes

When things go wrong.

Everything below has happened to a real person at least once. None of it is a permanent failure unless explicitly noted.

I lost my 6-word backup passphrase. What now?

Permanent. The passphrase is the only key to the encrypted backup blob; nobody — not us, not the wallet provider, not the network — can recover it. The encrypted bytes stay on the network forever, but they look like noise without the passphrase.

Make a new backup with a fresh passphrase from your current device, write it down on paper, and put it somewhere physical you trust. The old backup blob is harmless.

I lost my wallet seed phrase. What about my chain/it list?

Without the wallet, you can't sign the chainmark-derive message, and without that signature you can't reproduce the encryption key. The entries are still on Arbitrum but they look like random bytes.

If you backed up the chainmark key separately (Settings → chain/it → Back up chainmark key), you can paste it on a new device and decrypt without the wallet. If not — the list is gone, but your underlying public etches are still fetchable by their addresses.

"Sync chainmarks" fails with "Missing or invalid. request() chainId: eip155:42161".

Your wallet's WalletConnect session was paired against a different chain than Arbitrum One. The desktop client now switches the wallet to Arbitrum automatically before signing — if it still fails, the session was approved against networks that don't include Arbitrum at all.

Fix: in your wallet app, revoke etchit's pairing (in MetaMask: Settings → Connections → revoke), reconnect from etchit, and confirm Arbitrum One when the wallet asks which networks to approve.

The app is stuck on "Connecting…" or peers don't seem to attach.

The Autonomi peer-to-peer network is still a live thing — some bootstrap peers go offline. The app dials several in parallel and only needs one to attach.

If the spinner sticks: open Settings → Bootstrap peers, replace the list with the current published one (linked from the etchit GitHub README), tap Save & Reconnect.

A fetch returns nothing or hangs forever.

Two possible causes. Most common: transient peer routing issues — wait a moment and try again. The data is replicated; another peer almost certainly has it.

Less common: the address is malformed (wrong length, extra spaces, bad copy-paste). A valid etch address is exactly 64 lowercase hex characters with no 0x prefix.

My private etches disappeared after reinstalling the app.

Expected — private etch data maps live in the app's encrypted local storage, which Android wipes when you uninstall. They're not on any server and not on chain/it (only public etch addresses sync).

If you backed up to the network beforehand: Restore from backup with the address + 6-word passphrase. If you didn't back up: the data maps are gone, and the encrypted blobs on the network are unreachable without them.

Habit to build: back up after every batch of private etches you'd be sad to lose.
