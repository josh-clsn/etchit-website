fetchit is a read-only viewer for the Autonomi network. Give it any address and it fetches the content by hash and renders it — text, PDFs, images, audio, video, even full HTML pages with working scripts. No wallet, no account: fetching is always free. It's a reader, not a host — what it pulls is cached privately on your device, the way a browser caches a page you visited. The other half of etch/it: that one writes, this one reads.

// What it does

Give it an address. It figures out the rest.

Autonomi addresses don't say what they hold — so fetchit looks. It sniffs the bytes, picks a renderer, and shows you the thing. One paste, one screen.

01 / FETCH
Fetch anything.

Text and Markdown render inline. JSON pretty-prints. PDFs page through. Images, audio and video play in place. Archives list their contents. Anything else downloads as a file you can save. One input box, every content type the platform can read.

02 / RUN
Run real pages.

Point it at an HTML page on Autonomi and it runs like a page on the web — the full Fetch API, WebAssembly, media playback, range requests. Links between autonomi://<address> pages resolve in-app. A whole multi-page site can live on the network with no DNS and no origin server.

03 / DOWNLOAD
Whole files, then play.

Autonomi reassembles a file from every one of its chunks — so a track or a clip downloads in full before it plays; there's no progressive stream. Once it's down, it's cached: scrub and seek anywhere, instantly, out of the copy on your device. Slow the first time, free every time after.

04 / CACHE
Yours, cached.

What you fetch is kept in a private cache on your device — content-addressed, size-capped, evicted oldest-first, just like a browser cache. fetchit is a viewer, not a server: it doesn't host anyone's content, doesn't re-serve it, and holds nothing for you between sessions but that local cache. Clear it any time.

// The other half

It reads what etch/it writes.

An address from etchit works in fetchit, and vice versa — they're the same network. If fetchit is how you open things, etchit is how you put them there.

// The facts

No server. No catch. No fee to read.

01
0fee
Fetching is free. No wallet, no account — you only pay to write, never to read.
02
0host
No fetchit server. The app is a thin client straight to the public network.
03
B3addr
Addresses are content hashes — what comes back is byte-for-byte what was written, or it doesn't resolve.
04
∅stream
No streaming. Files arrive whole, then play. Range requests work — but only over the cached copy.
05
Rbuild
Reproducible builds. The APK is verifiable against the source in-tree.
06
OSS
Fully open source — the Android app and the shared native networking core, all in-tree.
