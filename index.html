One network — two apps.

Autonomi is a public, content-addressed network. Pay once to put something on it and it's reachable forever, by anyone with the address — no server, no account, nothing anyone (including us) can quietly take down. etch/it is how you write to it. fetch>it is how you read it. Two small apps, one idea: a link that can't rot.

// Two apps, equal billing

One writes. One reads.

They share a network and a design, nothing else. etchit puts things on Autonomi; fetchit takes them off. Use either, both, or neither — the address works from any client on the network.

The writer · publish
etch/it
Etch it once.
Fetch it forever.

Save text to Autonomi and share it with a single permanent address. Connect your own wallet, pay a small one-time fee to the storage peers, and what you etched stays reachable as long as the network runs — public, or private behind biometric auth on your device.

  • Text, notes, code, long-form — anything that fits as text
  • Public or private etches; private keys stay in the Android Keystore
  • Encrypted backups + chain/it sync across devices
  • Non-custodial · pay once, no subscription

The reader · view
fetch>it
Paste an address.
See what's there.

A read-only viewer for the network. Paste any Autonomi address and fetchit fetches it by content hash and renders it — text, PDFs, images, audio, video, even full HTML pages with working scripts. No wallet, no account: fetching is always free. A reader, not a host — what it pulls is cached privately on your device, like a browser cache.

  • Auto-detects what an address holds and renders it inline
  • HTML pages run with real web APIs — fetch, WebAssembly, media
  • Files download whole, then play / open / save — cached for offline
  • Free to read · open source · Android

// What's underneath

Built like infrastructure, not a service.

Neither app has a server. Both are thin clients over the public Autonomi network — addresses are derived from content with BLAKE3, payments settle on Arbitrum One, and the binaries are reproducible from the source you can read.

01
∞reach
Pay once to write. Stored as long as the network runs. No renewals, no expiry.
02
0accts
No signups, no email, no password. A wallet to write; nothing at all to read.
03
0custody
No etchit server, no etchit database. The apps never hold your data or keys.
04
B3addr
Addresses are content hashes — what you fetch is byte-for-byte what was written, or it doesn't resolve.
05
Rbuild
Reproducible builds. The APK we ship is verifiable against the code in-tree.
06
OSS
Both apps and the shared native networking core are fully open source.
